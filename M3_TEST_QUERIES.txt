SUMMARY OF IMPROVEMENTS

1. Stop Word Detection and Down-weighting
   - Identified corpus-specific common terms (uci, ics, informatics, computer, etc.)
   - Reduced IDF weight by 70% for these terms
   - Impact: Prevents common terms from dominating rankings

2. Query Expansion for Acronyms
   - Automatic expansion of common acronyms (AI, ML, HCI, NLP, etc.)
   - Searches for both acronym and full term
   - Impact: Improves recall for acronym queries

3. Course Code Normalization
   - Regex pattern to detect 'dept + number' format
   - Generates combined form (cs122) from spaced form (cs 122)
   - Impact: Handles various course code formats

4. Relaxed Boolean AND for Stop-Word Queries
   - Only requires intersection on meaningful terms
   - Falls back to union if intersection is empty
   - Impact: Prevents no-result scenarios

5. Enhanced Importance Boosting
   - Increased boost from 2.0x to 2.5x for title/heading terms
   - Impact: Stronger signal for page relevance

6. Original Term Weighting
   - 1.5x weight for user's actual query terms vs expanded terms
   - Impact: Maintains query intent while benefiting from expansion

7. Document Length Normalization
   - Penalty for documents with > 100 matching terms
   - Formula: score / (1 + log(total_terms/100))
   - Impact: Prevents long documents from dominating results
